# Тестовое задание SnapJoy Games

Версия Unity: 6000.2.6f2
Render Pipeline: URP
Система ввода: Input System Package

## Архитектура

В основе проекта лежит компонентно-ориентированный подход для расширяемости и слабой связанности.

#### Система взаимодействия

Логика ввода реализована в классе PlayerInteraction, там обработка фаз нажатий клавиши E.

Все что связано со взаимодействием завязано на интерфейсе IInteractable и абстрактном классе InteractableBase. Интерфейс обзывает реализовать метод GetInteractText(), который передает в UI текст, соответствующий предмету и всяким доп условиям. Для ключа и похожих предметов сделана система условий, когда для взаимодействия с предетом нужно выполнить условие(-я), например наличие предмета в руках (можно в теории написать любые другие условия).

От себя: дополнительный тип взаимодействия HackTerminal. Это мини-игра на тайминг, где нужно успеть нажать E в нужный момент.

#### Состояния игрока (State Machine)

У игрока есть три независимых состояния:
- ExplorationState: стандартный геймплей.
- InspectionState: блокирует перемещение и вращение камеры, включает мышь и дает крутить объект.
- DialogueState: фокусирует ввод на UI диалогов.

Такой подход гарантирует, что логика одного состояния не будет конфликтовать с другим (например, игрок не сможет уйти от NPC во время важного разговора).

#### Связанность

GameManager - это Generic синглтон, который просто собирает в себе ссылки на глобальные штуки, которые нужны всем: менеджеры, UI, компоненты игрока. Для того, чтобы не создавать лишние связанности между скриптами и уменьшить надобность писать новый код, большая часть взаимодействий между скриптами (особенно потенциально настраиваемая) сделана через UnityEvents.

#### Предметы и гнезда

Гнезда ItemSpcket не привязаны к типам - они могут хранить в себе любой Item. Любой Item можно осмотреть с плавным вращением предмета мышкой (а еще его названием и описанием), после чего взять в руку. Еще есть предметы: предмет записка с текстом и анимацией открытия, предмет часы, на которых можно смотреть время (ненастоящее).

#### Диалоговая система

Простая система нелинейных диалогов на основе SO, где можно вручную задать на какую реплику переключится текущая, для этого есть небольшой кастомный редактор. Имена собеседников выбираются с помощью enum (имя игрока из конфига и имя NPC из его инспектора)

#### Квестовая система

Простая система, позволяющая запустить квест на поиск рандомного предмета. Чтобы исключить отдельные предметы из поиска, у всех предметов есть галочка в инспекторе для этого. С диалогом система интегрирутся через UnityEvents

#### Механики окружения

Вентиль, который нужно удерживать для открытия двери, рычаги двух типов (одноразовые и многоразовые), двери. От себя: Теминал для взлома с мини-игрой, где нужно попасть в тайминг (это отдельный тип взаимодействия), немного настраиваемый.

#### Конфиги

Один глобальный конфиг - PlayerConfig со всеми настройками, связанными с игроком. И отдельные SO для данных диалогов с кастомным редактором. Все интерактивные предметы, объекты, менеджеры и элементы UI это префабы, так что их можно легко и отдельно друг от друга настроить.

## Структура проекта

- Common: глобальные системы (GameManager, SingletonBehaviour), общие визуальные ассеты и модели.
- InteractionSystem: базовая логика интерфейсов, условий (HasItemCondition) и типов взаимодействия (Hold, Toggle, OneTime).
- ItemsSystem: логика предметов, универсальных сокетов и уникальные скрипты объектов (PocketWatch, NoteItem).
- PlayerControl: контроллер, PlayerConfig (SO), машина состояний и центральный UI игрока.
- DialogueSystem: менеджер диалогов, UI реплик и кастомный редактор данных.
- QuestSystem: логика квестов и чекбокс-индикатор.
- EnvironmentObjects: скрипты физических объектов уровня (Door, Valve).
